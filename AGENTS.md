# Repository Instructions

These instructions apply to the Liquid repository. Global agent behavior,
source-control safety, secret handling, and generic validation discipline come
from the global Codex AGENTS file.

## Project Context

- Liquid builds a headless Raspberry Pi OS Lite image for a Raspberry Pi 3 A+.
- The image bakes this repo into `/home/artist/liquid`; the deployed checkout
  remains a normal Git repo for explicit runtime updates after Wi-Fi setup.
- The user-facing runtime entrypoint is `liquid`, a thin installed shim into
  `scripts/liquid`.
- Runtime renderer settings are local installation state in
  `/home/artist/liquid/.liquid/settings.env`; keep `.liquid/` untracked.
- The Rust renderer lives under `code/` and uses the package name `fluid_sim`.
- The terminal renderer is the default runtime path. The WS2812B LED matrix
  renderer is optional and builds behind the `led-matrix` feature.

## Repository Layout

- `README.md`: operator-facing setup, flashing, first-boot, runtime, Bluetooth,
  diagnostics, GitHub image build, local development, and LED matrix notes.
- `docs/ARCHITECTURE.md`: Pi runtime layout, boot behavior, Rust modules,
  terminal rendering, and LED matrix architecture.
- `docs/DEVELOPMENT_STEPS.md`: current state, validation checkpoints, and next
  real-hardware test work.
- `code/`: Rust simulation and renderers.
- `scripts/liquid`: main runtime/menu/update/install script.
- `scripts/*.sh`: workstation and Pi helper scripts.
- `image/`: rpi-image-gen config, layer, baked shell files, command shim,
  systemd-related files, and pre-image patching.
- `.github/workflows/ci.yml`: shell and terminal-renderer checks.
- `.github/workflows/build-image.yml`: Raspberry Pi image build and release
  asset publishing.

## Local Rules

- Treat the generated image as the deployment artifact; do not commit generated
  image work directories, compressed image assets, or release artifacts.
- Do not commit Wi-Fi credentials, SSH private keys, Bluetooth pairing secrets,
  local renderer settings, or Pi-specific host/network details.
- Keep runtime logic in the repo checkout, especially `scripts/liquid`; the
  installed `/usr/local/bin/liquid` should stay a thin shim.
- Preserve explicit update behavior. The Pi should not auto-pull from GitHub on
  boot.
- Preserve local installation state during migration/update flows: Wi-Fi,
  Bluetooth pairings, SSH setup, the Git checkout, and `.liquid/settings.env`.
- LED matrix work must respect the hardware notes in `README.md`: external 5V
  power, shared ground, SPI0 MOSI on BCM GPIO 10 / physical pin 19, and a logic
  level shifter recommended for WS2812B data.
- Run LED matrix commands only on a correctly wired Pi; do not treat them as
  general local workstation checks.

## Documentation

- Update `README.md` when changing operator workflows, flashing/download
  steps, first-boot behavior, runtime commands, Bluetooth/diagnostic behavior,
  local development commands, or LED wiring guidance.
- Update `docs/ARCHITECTURE.md` when changing image layout, runtime ownership,
  settings storage, boot behavior, Rust module responsibilities, or renderer
  architecture.
- Update `docs/DEVELOPMENT_STEPS.md` when completing, reordering, or adding
  validation checkpoints and next hardware/operator steps.

## Verification

For renderer/runtime code changes, prefer the validation set documented in
`docs/DEVELOPMENT_STEPS.md`:

```sh
cargo fmt --manifest-path code/Cargo.toml --check
cargo check --manifest-path code/Cargo.toml --no-default-features --example terminal
cargo check --manifest-path code/Cargo.toml --no-default-features --features led-matrix --example led_matrix
cargo check --manifest-path code/Cargo.toml --features window
bash -n scripts/liquid scripts/*.sh image/pre-image.sh image/files/usr/local/bin/liquid image/files/usr/local/sbin/liquid-*
zsh -n image/files/home/artist/.zshrc image/files/home/artist/.liquid-shell.zsh image/files/home/artist/.liquid-shell.d/*.zsh
git diff --check
```

For a bounded terminal smoke test:

```sh
scripts/liquid run --fixed-size --cols 40 --rows 20 --particles 500 --color cyan --charset dots --gravity-spin 0 --frames 5
```

CI also runs ShellCheck when available:

```sh
shellcheck scripts/liquid scripts/*.sh image/pre-image.sh image/files/usr/local/bin/liquid image/files/usr/local/sbin/liquid-*
```

## Build And Release

- The `Build image` workflow runs on pushes to `main` that affect image,
  runtime, Pi scripts, or the workflow itself, and it can also run manually.
- The image workflow pins `rpi-image-gen` to `v2.6.0`.
- Release image assets are generated by CI and must remain out of git.
- Manual release tags should follow the workflow examples, such as
  `liquid-rpi3-lite-2026-05-25`.

## Known Caveats

- The current useful docs are `README.md`, `docs/ARCHITECTURE.md`, and
  `docs/DEVELOPMENT_STEPS.md`; older product/spec doc names may appear in stale
  copied instructions but do not exist in this repo.
- Real LED matrix validation is still pending for the first 8x8 WS2812B matrix.
- Bluetooth pairing improvements depend on observing the current dialog wrapper
  failure mode on real hardware.
